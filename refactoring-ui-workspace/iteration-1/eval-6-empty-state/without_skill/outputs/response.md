# Designing Empty States

A blank white area is a missed opportunity. Empty states are actually prime real estate for guiding users and setting expectations.

## Core Principles

### 1. Explain What Goes Here
Show users what the section is *for*. A brief headline like "No projects yet" paired with a short description ("Create a project to start organizing your work") makes the purpose immediately clear.

### 2. Provide a Call to Action
The empty state should include a direct action button — the primary thing a user can do to fill this space. Don't make them hunt for it. Place a prominent button like "Create your first project" right in the empty state itself.

### 3. Use an Illustration or Icon
A simple illustration or icon makes the empty state feel intentional rather than broken. It signals "this is expected" rather than "something went wrong." Avoid overly complex illustrations — a clean, relevant graphic is enough.

## Visual Design Tips

- **Don't use full-page white** — add subtle background contrast or center the empty state content in the column to make it look deliberate.
- **Keep it compact** — the empty state shouldn't dominate the screen. Center the message vertically and horizontally in the section, using modest padding.
- **Use muted/secondary text color** for the description — it reads as context, not as primary content.
- **Style the CTA like your primary button** — consistent with the rest of your UI, not a special one-off style.

## Tone

Match your app's personality. A playful app can use a friendly illustration and casual copy. A professional tool should be clean and direct. Either way, avoid generic filler text like "Nothing here yet" with no guidance.

## Example Structure

```
[ Illustration / Icon ]

  No items found

  You haven't added anything yet.
  Get started by creating your first item.

  [ + Add Item ]  ← primary action button
```

## What to Avoid

- **Blank space with no explanation** — users may think it's a bug.
- **Error-like styling** — empty ≠ broken.
- **Too much text** — keep it scannable.
- **Hiding the CTA** — the action to fill the space should be immediately visible.

## Summary

A well-designed empty state = illustration/icon + short headline + brief description + clear CTA. It turns a dead end into a starting point.
