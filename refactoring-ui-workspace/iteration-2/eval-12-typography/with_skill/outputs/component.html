Design Systems

Typography Makes or Breaks Your Interface

A well-considered type system isn't about picking a beautiful font. It's about making deliberate decisions at every scale step so that users always know what to read first — and what to ignore.

Sarah Chen · June 12, 2025 · 8 min read
Chapter One

Start With a Scale, Not a Font

Most designers reach for a typeface first. That's backwards. Before you pick Inter or Playfair Display, establish the size constraints your type will live within. A scale is a fixed set of font sizes — and the discipline is to use only those sizes, without exception.

When you have a scale, every sizing decision becomes a lookup rather than a judgment call. You're not asking "should this be 17px or 18px?" You're asking "is this base or large?" That shift — from open-ended to constrained — is what makes design systems actually work in practice.

"Every time you size text outside your scale, you're spending design debt you'll eventually have to repay — usually during a visual audit at 2 am."

The constraint also prevents a common failure mode: creeping intermediate sizes. Without a scale, codebases accumulate a dozen sizes differing by a pixel or two, each added because one specific element "just needed a little more presence." The result is visual noise that reads as inconsistency — even if no single choice was wrong.

Chapter Two

Hierarchy Needs Three Tools, Not One

Relying on font size alone to create hierarchy forces extreme choices — headlines so large they feel aggressive, body text so small it strains the eye. Size is one lever. The system works only when you use all three.

Font Weight

Weight is the most underused tool in UI typography. A 600-weight heading at 20px can feel more prominent than a 400-weight heading at 24px. Use 400–500 for most text and reserve 600–700 for elements that genuinely need to command attention. Avoid weights below 400 in UI — at small sizes they collapse into illegibility.

Color and Contrast

A three-tier color system handles most hierarchy needs on its own. Primary text in near-black for headlines and key data. Secondary text in mid-grey for supporting content, dates, and metadata. Tertiary text in light grey for captions and helper copy. The principle is to de-emphasize rather than always trying to amplify.

Proportional Line Height

Large headings need tight line height — 1.1 to 1.2. Body text at normal column widths needs 1.5 to 1.65. Wide columns need even more space between lines because the eye has further to travel to find the start of the next one. There is no single correct value; it always scales with context.

Closing

The System Is the Work

A type system is never truly finished. New content surfaces edge cases; new breakpoints reveal that a heading that looked elegant on a 1440px monitor becomes overwhelming on a 375px screen. The scale gives you a foundation — the work is in applying it consistently enough that the system becomes invisible and the content speaks for itself.

What separates polished UI from amateur work is rarely the font choice. It's the decision to use 30px instead of 28px because 30 is on the scale and 28 is not. It's the patience to de-emphasize secondary text instead of reaching for another bold override. It's choosing proportional line-height for each context instead of setting everything to 1.5 and calling it done.

The constraints are the craft.
